# Projects Engineering Log

## Project 1 -  Shopping Cart v1

#### EL 1

Before starting any project, spend a few minutes creating a blueprint. Break the project into small, independent features before writing code. Build and test one feature at a time instead of trying to code the entire project at once.

---

#### EL 2

After finishing the blueprint, ask yourself these three engineering questions:

- 1.What information must my program remember? (Data)

- 2.What actions can the user perform? (Features)

- 3.What is the normal journey from start to finish? (Flow)

---

#### EL 3

Let the loop control repetition, not individual features.

---

#### EL 4

Never assume the user will always do the right thing. Design for mistakes, not just success.

---
#### EL 5

Normalize data before validating it.

---
#### EL 5

When possible, fic the small mistakes where they happen instead of making the user restart the whole process.

---

#### EL 6

Whenever you use prompt(), Ask do need to clean the input using methods like trim, parseInt, toUpperCase.
